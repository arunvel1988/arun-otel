3. Flask Application Creating Logs

app.py 


from flask import Flask

import logging



app = Flask(__name__)



logging.basicConfig(

level=logging.INFO,

format="%(levelname)s service=cart %(message)s"

)



@app.route("/checkout")

def checkout():



    logging.info(

    "checkout completed user=101 trace_id=abc123"

    )



    return "success"





app.run(

host="0.0.0.0",

port=5000

)

8. Loki Indexing Model

Very important concept. Loki does NOT index complete log text.

Traditional logging: 


Elasticsearch:


Indexes:


Every word


Every field


Every log
Loki: 


Indexes ONLY labels


Stores logs compressed

9. What Are Labels?

Labels describe logs.

Example: 


{


app="shopping",


service="cart",


environment="production"


}
These are indexed. Good labels: 


service


environment


cluster


namespace
Bad labels: 


user_id


order_id


request_id
Because they create millions of streams.

10. Loki Streams

A stream is a unique combination of labels.

Example: 


Stream 1:


{


service="cart",

env="prod"

}



contains:



log1


log2


log3
Another: 


{


service="payment",

env="prod"

}


creates another stream.

11. LogQL Query Language

LogQL is used to search Loki logs.

Find service logs: 


{service="cart"}
Find errors: 


{service="cart"}

|= "ERROR"
Search checkout: 


{service="cart"}

|= "checkout"

12. JSON Log Query

JSON Log: 


{


"service":"cart",

"status":500,

"user":"101"

}
Query: 


{service="cart"}

| json

| status=500

13. Regex Query

Log: 


payment failed user=101
Query: 


{service="payment"}


| regexp "user=(?P<user>.*)"
Output: 


user=101

14. Logs To Metrics

Loki can calculate metrics from logs.

Example: 


How many errors in 5 minutes?
LogQL: 


count_over_time(

{service="payment"}

|= "ERROR"

[5m]

)

15. Correlating Logs With Traces

Logs tell what happened. Traces tell where it happened.

Example: 


User checkout request

abc123

{


message:"payment failed",


trace_id:"abc123"


}
Tempo trace: 


trace_id abc123



checkout


   |


payment


   |


database
Same trace_id connects both.
