Grafana Mimir Cardinality Explained Simply

First: What does Mimir store?

Mimir does not store just metric names. Mimir stores time series.

A time series means: 

Metric Name
+
Labels
+
Label Values
Example metric: 
http_requests_total
With labels: 
http_requests_total{
 service="cart",
 method="GET",
 status="200"
}
This complete combination becomes ONE SERIES.

Example: Low Cardinality

Imagine shopping application: 
http_requests_total{
 service="cart",
 status="200"
}

http_requests_total{
 service="payment",
 status="500"
}
Only few combinations. Result: 
2 active series
Mimir is happy.

Example: High Cardinality Problem

Developer adds user_id label: 
login_total{
 user_id="1001"
}

login_total{
 user_id="1002"
}

login_total{
 user_id="1003"
}

...

login_total{
 user_id="1000000"
}
Now what happens? 
1 metric
+
1 million users

=

1 million time series
Mimir has to keep track of all these series.

Why is High Cardinality Bad?

Every active series needs:
  • Memory in Ingester
  • Index information
  • CPU for queries
  • Storage in S3 blocks

Mimir Cardinality Limits

Example configuration: 
limits:

 max_global_series_per_user: 1000000
Meaning: 

Tenant A

maximum allowed active series:

1 Million
After crossing: 

Incoming new series

        |
        v

        X


Rejected

Important Point

Mimir DOES NOT remove bad labels automatically. Example: 
payment_total{
 user_id="12345"
}
Mimir will not change it into: 
payment_total{}
Mimir only says: 

Accept

or

Reject

Correct Production Solution

Fix before Mimir using Alloy. Before: 
checkout_total{
 user_id="789",
 region="india"
}
Remove user_id: 
checkout_total{
 region="india"
}
Flow: 

Application

     |
     v

Alloy

(drop labels)
(drop metrics)

     |
     v

Mimir

(limit protection)

     |
     v

S3

Alloy Example

Remove dangerous label: 
prometheus.relabel "cleanup" {

 rule {

   action = "labeldrop"

   regex = "user_id"

 }

 forward_to = [
   prometheus.remote_write.mimir.receiver
 ]
}
Now Mimir receives clean metrics.

Final Understanding

Question Answer
Does Mimir know cardinality? YES
Can Mimir count active series? YES
Can Mimir stop too many series? YES
Can Mimir remove bad labels? NO
Where to fix cardinality? Application / Alloy
