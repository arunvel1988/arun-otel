1. Flask Application Without Monitoring

This is a simple shopping application. 


from flask import Flask


app = Flask(__name__)


@app.route("/cart")
def cart():

    return "Product added"


@app.route("/checkout")
def checkout():

    return "Order completed"



app.run(
host="0.0.0.0",
port=5000
)

Problem: We don't know: - how many cart requests happened - how many checkout happened - API performance

2. Add OpenTelemetry Metrics SDK

Install dependencies: 


pip install flask

pip install opentelemetry-sdk

pip install prometheus-client

Now we create metrics inside Flask.

3. Custom Business Metrics Example

Business requirement: Count how many products users add into cart. 


from flask import Flask

from prometheus_client import Counter
from prometheus_client import generate_latest



app = Flask(__name__)



cart_counter = Counter(
    "cart_count",
    "Total cart additions"
)



checkout_counter = Counter(
    "checkout_count",
    "Total checkout completed"
)




@app.route("/cart")
def cart():

    cart_counter.inc()

    return "Added to cart"




@app.route("/checkout")
def checkout():

    checkout_counter.inc()

    return "Checkout success"





@app.route("/metrics")
def metrics():

    return generate_latest()



app.run(
host="0.0.0.0",
port=5000
)

Generated Metrics Output

When user calls: 


curl localhost:5000/cart

curl localhost:5000/cart

curl localhost:5000/checkout

Metrics endpoint: 


curl localhost:5000/metrics


OUTPUT:


cart_count_total 2


checkout_count_total 1

6. Grafana Alloy Collector

Alloy finds Flask metrics endpoint and scrapes it. 


Flask exposes:


http://flask:5000/metrics



Alloy reads this every few seconds
Alloy Configuration: 

prometheus.scrape "flask" {


targets = [

{

"__address__"="flask:5000"

}

]



forward_to = [

prometheus.remote_write.mimir.receiver

]


}
