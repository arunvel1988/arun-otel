Day 4 - Distributed Tracing Fundamentals

Flask Microservices + OpenTelemetry Tracing

1. Why Distributed Tracing?

Modern applications have many services. One user request travels through multiple applications. 



Customer clicks checkout


          |


      API Gateway


          |


      Cart Service


          |


    Payment Service


          |


       Database
Problem: 

Checkout is slow


Question:


Which service caused delay?


Cart?


Payment?


Database?
Tracing answers this.

2. Simple Flask Microservice Example

Two services: 



checkout-service


        calls


payment-service
checkout.py 

from flask import Flask

import requests



app = Flask(__name__)



@app.route("/checkout")

def checkout():



    response = requests.get(

       "http://payment:5001/pay"

    )



    return "checkout completed"




app.run(

host="0.0.0.0",

port=5000

)
payment.py 


from flask import Flask


app=Flask(__name__)



@app.route("/pay")

def pay():


    return "payment success"




app.run(

host="0.0.0.0",

port=5001

)

3. What Is A Trace?

A trace represents the complete journey of one request.

Example: 


User Checkout Request


Trace ID:


4bf92f3577b34da6



               TRACE


------------------------------------------------


Checkout Service


Payment Service


Database Query
All operations share the same Trace ID.

4. What Is A Span?

A span represents one operation inside a trace.

Example: 


Trace ID: abc123




Checkout Request

       |

       |

       +---- Span 1

       |
       |


Payment Call

       |

       +---- Span 2


       |


Database Query


       +---- Span 3
Every span contains: 


span_id


start time


end time


duration


attributes


events

5. Span Timeline

Tracing tools show execution time.

Example: 


Trace: checkout



checkout-service

|-----------------------|

0ms                    800ms




payment-service

      |-------------|

      200ms       700ms




database

            |----|

            400ms
Now developer knows database was slow.

6. Trace ID And Span ID

Trace ID connects the complete request. Span ID identifies individual work.

Example: 


TRACE


trace_id = abc123




SPAN 1


service=checkout

span_id=111




SPAN 2


service=payment

span_id=222



SPAN 3


service=database

span_id=333

7. OpenTelemetry Flask Instrumentation

Install: 


pip install opentelemetry-sdk

pip install opentelemetry-instrumentation-flask


pip install opentelemetry-instrumentation-requests
Code: 


from flask import Flask


from opentelemetry.instrumentation.flask import FlaskInstrumentor



app=Flask(__name__)



FlaskInstrumentor().instrument_app(app)




@app.route("/checkout")


def checkout():



    return "success"
Now spans are automatically created.

8. Attributes

Attributes are extra information attached to spans.

Example: 


Span:


checkout




Attributes:



user.id = 100


cart.items = 5


region = india
Python: 


from opentelemetry import trace



span = trace.get_current_span()



span.set_attribute(

"user.id",

"100"

)



span.set_attribute(

"cart.items",

5

)

11. Context Propagation

Problem: How does payment service know checkout trace id?

Without propagation: 


checkout trace = abc


payment trace = xyz
They look unrelated. With propagation: 


checkout trace = abc


payment trace = abc

12. W3C TraceContext

OpenTelemetry follows W3C standard headers.

HTTP request contains: 


traceparent:


00-4bf92f3577b34da6-00f067aa0ba902b7-01
Meaning: 


version


trace id


span id


flags
Flow: 


Checkout Service



adds header:



traceparent



        |



Payment Service



reads header



continues same trace
