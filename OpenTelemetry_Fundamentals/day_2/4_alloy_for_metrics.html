Grafana Alloy for Metrics

Discovering and scraping application metrics

Why Do We Need Grafana Alloy?

Imagine we started with a small ecommerce application. The application exposes metrics and Prometheus collects them.

Ecommerce Flask Application
↓
Prometheus
↓
Grafana Dashboard

This works when there is only one application. But real companies have hundreds of services.

Cart Service
↓
Payment Service
↓
Order Service
↓
Inventory Service

Now problems start.

  • Who will discover all applications?
  • Who will collect all metrics?
  • Who will add common labels?
  • Who will filter unnecessary metrics?
  • Who will send data to different backends?

Grafana Alloy solves these problems.

What is Grafana Alloy?

Grafana Alloy is an observability collector. It runs between applications and monitoring systems.

Applications
↓
Grafana Alloy Receive Process Export
↓
Prometheus / Mimir

Applications only create metrics. Alloy takes responsibility for moving and processing them.

Without Alloy vs With Alloy

Without Alloy With Alloy
Every monitoring backend connects directly to applications Single collection layer between apps and backends
Hard to manage many services Central configuration
No common processing Filtering, labels and transformations
Difficult in Kubernetes Automatic discovery support

What is Metric Discovery?

Discovery means automatically finding applications that expose metrics.

Example: Today ecommerce has 10 containers. Tomorrow auto scaling creates 100 containers. Alloy should automatically find them.
New Application Started
↓
Alloy Discovers It
↓
Starts Collecting Metrics

What is a Metrics Endpoint?

Applications expose their measurements through an HTTP endpoint. Usually:

/metrics
Example:
http://cart-service:5000/metrics Response: cart_count_total 500 checkout_count_total 300 payment_error_total 20

This text format is called Prometheus metrics format.

What is Scraping?

Scraping means Alloy periodically connects to applications and reads metrics.

Every 10 Seconds
↓
Alloy calls /metrics
↓
Collects latest values
Example:
10:00 AM cart_count_total = 100 10:10 AM cart_count_total = 600 Alloy observes the growth

Alloy Prometheus Scrape Example

prometheus.scrape "ecommerce" { targets = [ { __address__ = "flask-app:5000" } ] forward_to = [ prometheus.remote_write.mimir.receiver ] }
Explanation:
  • Connect to Flask application
  • Read Prometheus metrics
  • Forward them to storage
