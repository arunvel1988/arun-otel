prometheus.yml Explained

The configuration file that tells Prometheus what to monitor

1. What is prometheus.yml?

Prometheus does not automatically know your applications. prometheus.yml tells Prometheus:

What should I monitor? Where are the applications? How frequently should I collect metrics? What rules should I load? Where should I send metrics?

2. Basic prometheus.yml Structure

global: scrape_interval: 15s scrape_configs: - job_name: ecommerce static_configs: - targets: - app:5000

3. Global Section

Global settings apply to all jobs unless overridden.

global: scrape_interval: 15s evaluation_interval: 15s
scrape_interval How often Prometheus collects metrics evaluation_interval How often alert rules execute

4. Scrape Configs

This is the most important section. It defines what Prometheus monitors.

scrape_configs: - job_name: ecommerce
A job is a group of targets. Examples: job = frontend job = database job = kubernetes job = node-exporter

5. Targets

Targets are actual applications exposing metrics.

static_configs: - targets: - ecommerce:5000 - payment:5000
Prometheus calls: http://ecommerce:5000/metrics http://payment:5000/metrics

6. Complete Scrape Flow

prometheus.yml | v Find target ecommerce:5000 | v HTTP GET /metrics | v Store result in TSDB

7. Labels in prometheus.yml

Labels add extra information to metrics.

static_configs: - targets: - app:5000 labels: environment: production team: payments
Result:
orders_total{ environment="production", team="payments" }

8. Multiple Jobs Example

scrape_configs: - job_name: application static_configs: - targets: - app:5000 - job_name: linux-server static_configs: - targets: - node-exporter:9100

9. Alert Rules Configuration

Prometheus can load alert files.

rule_files: - alerts.yml
Example alert:
CPU usage > 90% | v Create alert

10. Remote Write

Remote write sends Prometheus metrics somewhere else. Example: Mimir.

remote_write: - url: http://mimir:9009/api/v1/push
Prometheus | | remote_write v Mimir (long term storage)

11. Full Production Example

global: scrape_interval: 15s rule_files: - alerts.yml scrape_configs: - job_name: ecommerce static_configs: - targets: - app:5000 labels: env: prod remote_write: - url: http://mimir/api/v1/push

12. Summary

Section Purpose
global Default timing settings
scrape_configs What Prometheus monitors
targets Application endpoints
labels Add metadata
rule_files Alert definitions
remote_write Send metrics externally
