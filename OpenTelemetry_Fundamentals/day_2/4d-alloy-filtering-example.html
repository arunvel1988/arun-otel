How Metric Filtering Works inside Alloy

Real example using Ecommerce application

Where Exactly Filtering Happens?

Filtering happens inside Alloy after metrics are collected but before they are sent to storage.

Flask Ecommerce App
creates metrics
↓
Alloy Scrape
/metrics collected
↓
FILTERING STAGE
prometheus.relabel
↓
Prometheus / Mimir
stores clean metrics

Step 1: Application Creates Many Metrics

Your ecommerce Flask app exposes:

/metrics output: cart_count_total 500 checkout_count_total 200 payment_failed_total 20 debug_cart_test_total 999 local_developer_metric 123 python_gc_collection_total 400

Everything is currently visible. Alloy has not filtered anything yet.

Step 2: Alloy Scrapes Metrics

Alloy first collects everything.

prometheus.scrape "ecommerce" { targets = [ { __address__="flask-app:5000" } ] forward_to = [ prometheus.relabel.filter.receiver ] }

Important: Metrics are NOT sent directly to storage. They first go to relabel filtering.

Step 3: Filtering Rule

Now Alloy decides what should be removed.

Example: Remove all debug metrics.
prometheus.relabel "filter" { rule { source_labels = ["__name__"] regex = "debug_.*" action = "drop" } forward_to = [ prometheus.remote_write.mimir.receiver ] }
Explanation:
  • __name__ means metric name
  • regex finds matching metrics
  • debug_.* means anything starting with debug
  • action drop removes it

Before Filtering vs After Filtering

Before Alloy Filtering

cart_count_total checkout_count_total payment_failed_total debug_cart_test_total local_developer_metric

After Alloy Filtering

cart_count_total checkout_count_total payment_failed_total

Only useful metrics reach Prometheus and Mimir.

Dropping Labels Example

Sometimes the metric is useful but one label is dangerous.

Before:
checkout_total{ service="checkout", user_id="12345" }
Problem:

Millions of users create millions of time series.

Alloy removes only user_id:
rule { action = "labeldrop" regex = "user_id" }
After:
checkout_total{ service="checkout" }

Remember

Filtering location: Inside Alloy pipeline Before filtering: Application → Alloy → Raw metrics After filtering: Application ↓ Alloy scrape ↓ Alloy relabel/filter ↓ Prometheus/Mimir Goal: Store only useful metrics
