Prometheus Storage Internals

How Prometheus stores millions of metrics efficiently

1. What Does Prometheus Store?

Prometheus stores numeric values changing over time. Each value is called a sample.

Example: orders_total{service="checkout"} 10:00 -> 100 10:01 -> 150 10:02 -> 250 Prometheus stores: metric name + labels + timestamp + value

2. Where is Data Stored?

Prometheus has its own database called TSDB. Time Series Database.

Default directory: /prometheus

Inside Docker container:

docker exec -it prometheus sh cd /prometheus ls

3. Prometheus Storage Layout

/prometheus ├── wal │ │ Write Ahead Log │ Recent incoming samples │ ├── chunks_head │ │ Current active data │ ├── 01HFABCDE12345 │ │ Completed TSDB block │ │ ├── chunks │ │ │ │ compressed metric values │ │ │ ├── index │ │ │ │ label search database │ │ │ └── meta.json │ └── queries.active

4. Write Path

When a new metric arrives this happens.

Application Metric orders_total 100 | v Prometheus Memory (Head Block) | v WAL File (crash protection) | v Compressed Block (permanent storage)

5. What is WAL?

WAL means Write Ahead Log. It protects Prometheus from losing recent data.

New sample arrives | v Write immediately to WAL | v Store in memory If Prometheus crashes: Restart | Read WAL | Recover data

6. What are Chunks?

Chunks contain the real metric values. They are compressed to save disk space.

Before compression: 10:00 = 100 10:01 = 105 10:02 = 110 After compression: small binary chunk file

7. What is Index?

Index helps Prometheus quickly find metrics.

Query: orders_total{service="payment"} Prometheus checks: index | finds matching series | reads chunks

8. Query Flow

Grafana Query | v PromQL Engine | +------ Memory Head | +------ TSDB Blocks | v Result

9. Docker Persistence

Without volume, deleting the container deletes metrics.

docker compose down -v Data removed

Production style:

prometheus: volumes: - prometheus-data:/prometheus volumes: prometheus-data:
