Imagine we have a shopping application. 


API:

/cart


/checkout



User says:

"My payment failed"



Question:


What happened?

When happened?

Which service failed?

Logs help investigation

2. Flask Without Proper Logging

Normal application: 


from flask import Flask


app = Flask(__name__)



@app.route("/checkout")

def checkout():


    print("checkout failed")


    return "error"



app.run(port=5000)
Problem: 

checkout failed
We don't know: 

Which user?

Which request?

Which service?

Which trace?

3. Unstructured Logs

Plain text logs are called unstructured logs.

Example: 

ERROR checkout failed for user john
Problems: 

Hard to search

Hard to filter

Hard for machines to understand

4. Structured JSON Logging

Modern microservices write JSON logs.

Example: 

{

 "level":"ERROR",

 "service":"cart-service",

 "message":"checkout failed",

 "user":"john",

 "status":500

}
Benefits: 

Easy searching


Find:

status=500


Find:

service=cart-service

5. Flask JSON Logging Example


import logging

import json



logger=logging.getLogger()

logger.setLevel(logging.INFO)




@app.route("/cart")

def cart():


    log={

      "event":"cart_added",

      "user":"arun",

      "service":"cart"

    }



    logger.info(json.dumps(log))



    return "added"
Output: 

{

"event":"cart_added",

"user":"arun",

"service":"cart"

}

6. Contextual Logging

One user request travels through many services. We need one ID to connect everything.

Flow: 

User Request


     |


Flask


     |


Payment Service


     |


Database
Without Trace ID: 

cart completed


payment failed


database timeout
With Trace ID: 

{

trace_id:"abc123",

service:"cart",

message:"cart created"

}



{

trace_id:"abc123",

service:"payment",

message:"payment failed"

}
Now we know both logs belong to same request.

7. What Is span_id?

A trace contains multiple operations. Each operation has a span.

Example: 

Trace ID: 12345


       Checkout Request


              |


        ----------------


        |              |


   DB Query        Payment API



Span ID 1       Span ID 2

8. OpenTelemetry Logging API

Install: 

pip install opentelemetry-sdk


pip install opentelemetry-api


pip install opentelemetry-instrumentation-flask
Flask Example: 


from flask import Flask


from opentelemetry.instrumentation.flask import FlaskInstrumentor



app=Flask(__name__)



FlaskInstrumentor().instrument_app(app)




@app.route("/checkout")

def checkout():



    logger.info(

       "checkout started"

    )



    return "success"

9. Adding Attributes To Logs

Attributes give more information.

Example: 

logger.info(

"payment success",


extra={

"user_id":"101",

"order_id":"500",

"amount":"2000"

}

)
Log: 

{

message:"payment success",

user_id:"101",

order_id:"500",

amount:"2000"

}

10. Microservice Logging Best Practices


Always use JSON logs


Always include service name


Always include trace_id


Never log passwords


Add request id


Use proper log levels



INFO

WARN

ERROR
