1. Why Alloy For Logs?

Applications generate thousands of logs. We need something to: 


Collect logs

Understand logs

Add labels

Filter unwanted data

Send logs to storage

Grafana Alloy works as the log collector.

2. Flask Application Generating Logs

app.py 


from flask import Flask

import logging



app = Flask(__name__)



logging.basicConfig(

level=logging.INFO,

format="%(levelname)s %(message)s"

)



@app.route("/checkout")

def checkout():



    logging.info(

    "checkout completed user=101 amount=500"

    )



    return "success"





app.run(

host="0.0.0.0",

port=5000

)
Output: 

INFO checkout completed user=101 amount=500

3. Alloy Log Receivers

Receivers decide where Alloy collects logs from.

3.1 File Log Receiver

Application writes logs into files. Example: 


/var/log/app.log
Log: 

ERROR payment failed user=200
Alloy: 

local.file_match "app" {


path_targets = [

{

__path__="/var/log/app.log"

}

]

}



loki.source.file "app"{


targets = local.file_match.app.targets


forward_to=[

loki.write.default.receiver

]


}
Flow: 

app.log

   |

Alloy reads file

   |

Loki

3.2 Docker Log Receiver

Containers write logs automatically.

Check: 

docker logs flask-container
Example: 

INFO checkout completed
Alloy: 

discovery.docker "containers" {


host="unix:///var/run/docker.sock"


}



loki.source.docker "logs"{


host="unix:///var/run/docker.sock"


targets=discovery.docker.containers.targets



forward_to=[

loki.write.default.receiver

]


}

3.3 Journald Receiver

Linux system services store logs in journald.

Example: 

systemctl status nginx


journalctl -u nginx
Alloy: 


loki.source.journal "linux"{



forward_to=[


loki.write.default.receiver


]


}

4. Log Processing

Raw logs are usually messy. Processors convert them into useful data.

Raw: 

INFO checkout user=101 amount=500
Wanted: 

level=INFO

event=checkout

user=101

amount=500

5. Regex Parsing Processor

Regex extracts values from normal text logs.

Input: 

ERROR payment_failed user=55
Regex: 

(?P<level>ERROR|INFO)

(?P<event>\w+)

user=(?P<user>\d+)
Output: 

level = ERROR

event = payment_failed

user = 55
Alloy: 


stage.regex {


expression =

"(?P<level>\\w+) (?P<msg>.*)"

}

6. JSON Parsing Processor

Modern apps produce JSON logs. Input: 

{

"level":"ERROR",

"user":"101",

"message":"payment failed"

}
Alloy: 

stage.json {


expressions={


level="level"


user="user"


message="message"


}


}
After: 

level=ERROR

user=101

message=payment failed

7. Severity Mapping

Different apps use different names. Normalize them.

Example: 

warning

WARN

warn
Convert: 

severity = WARN
Alloy: 

stage.labels {


values={


severity="level"


}


}

8. Export Logs To Loki

After processing, Alloy sends logs to Loki.

Config: 


loki.write "default"{


endpoint {


url="http://loki:3100/loki/api/v1/push"


}


}
