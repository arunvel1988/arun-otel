Application Metrics Instrumentation

Understanding OpenTelemetry Metrics SDK, Automatic Metrics, and Custom Business Metrics using Flask examples.

What is Metrics Instrumentation?

Metrics instrumentation means adding metric collection capability into applications using OpenTelemetry Metrics SDK.

  • Collect application performance metrics
  • Collect infrastructure metrics
  • Monitor runtime behavior
  • Track business KPIs
  • Enable dashboards and alerts
Examples: API Request Count Checkout Latency Cart Count CPU Usage Database Query Time

Metrics Collection Flow

Using OpenTelemetry Metrics SDK

OpenTelemetry Metrics SDK allows developers to create counters, gauges, and histograms.

pip install opentelemetry-sdk pip install opentelemetry-api pip install opentelemetry-exporter-otlp
from opentelemetry.metrics import get_meter meter = get_meter("ecommerce-app")
Component Purpose
Meter Main metrics object
Counter Counts events
Gauge Current value measurement
Histogram Latency distribution measurement

Automatic Metrics

OpenTelemetry can automatically generate runtime, HTTP, database, and gRPC metrics.

Runtime Metrics

CPU usage, memory usage, garbage collection metrics.

HTTP Metrics

Request count, latency, status codes.

Database Metrics

Query duration, connection count, DB latency.

gRPC Metrics

RPC duration, request count, RPC failures.

Flask Automatic Metrics Example

from flask import Flask from opentelemetry.instrumentation.flask import FlaskInstrumentor app = Flask(__name__) # Enable automatic instrumentation FlaskInstrumentor().instrument_app(app) @app.route("/") def home(): return "Metrics Enabled" if __name__ == "__main__": app.run(port=5000)
Automatically Generated Metrics: http.server.duration http.server.request.count http.server.active.requests

Custom Business Metrics

Business metrics track domain-specific KPIs important for the organization.

  • Cart Count
  • Checkout Rate
  • Orders Per Minute
  • Payment Success Rate
  • Inventory Availability

Custom Counter Metric Example

Track total checkout requests using Counter metrics.

from flask import Flask from opentelemetry.metrics import get_meter app = Flask(__name__) meter = get_meter("ecommerce-app") checkout_counter = meter.create_counter( name="checkout_requests_total", description="Total checkout requests" ) @app.route("/checkout") def checkout(): checkout_counter.add(1) return "Checkout Completed" if __name__ == "__main__": app.run(port=5000)
Metric Generated: checkout_requests_total = 10500

Custom Gauge Metric Example

Track current active carts in the application.

from opentelemetry.metrics import get_meter meter = get_meter("ecommerce-app") active_cart_gauge = meter.create_observable_gauge( name="active_cart_count", description="Current active shopping carts" ) # Example Value active_cart_count = 320
Metric Generated: active_cart_count = 320

Custom Histogram Metric Example

Histograms help measure request latency distribution.

from opentelemetry.metrics import get_meter meter = get_meter("ecommerce-app") checkout_latency = meter.create_histogram( name="checkout_latency", description="Checkout API latency" ) # Record latency checkout_latency.record(250)
Latency Distribution: 0-100 ms 100-300 ms 300-500 ms 500+ ms
