LogQL Querying in Loki

Finding problems inside millions of logs

What is LogQL?

LogQL is the query language used by Loki.

Just like Prometheus uses PromQL for metrics, Loki uses LogQL for logs.

Prometheus: PromQL → Query metrics Loki: LogQL → Query logs

Why Do We Need LogQL?

Imagine ecommerce logs:

10 million logs today checkout logs payment logs database logs cart logs

Questions:

  • Show only payment errors
  • Find failed orders
  • Count checkout failures
  • Find slow API calls

LogQL helps answer these questions.

Basic Loki Query

First select logs using labels.

{service="payment"}
Meaning:
Go to Loki Index
↓
Find streams where service=payment
↓
Return payment logs

Multiple Label Search

Example:
{ service="payment", environment="prod" }
Meaning:
Give me logs where: service is payment AND environment is production

Searching Text Inside Logs

Labels find streams. Then we search inside messages.

Example:
{service="payment"} |= "failed"
Meaning:
Step 1: Find payment logs Step 2: Inside those logs find word failed

Exclude Text

Remove unwanted logs.

{service="payment"} != "success"
Meaning:
Show payment logs but remove logs containing success

Regex Searching

Regex allows pattern matching.

Example:
{service="checkout"} |~ "ERROR|FAILED"
Meaning:
Find checkout logs containing: ERROR or FAILED

JSON Log Querying

Modern applications create JSON logs.

Example log:
{ "service":"payment", "user_id":"100", "status":"failed" }
LogQL:
{service="payment"} | json | status="failed"
Meaning:
  • Select payment logs
  • Parse JSON
  • Filter status field

Finding User Journey

Using trace id from OpenTelemetry.

{service="checkout"} | json | trace_id="abc123"
Result:
Checkout started Payment called Database timeout Checkout failed

Counting Logs

LogQL can also behave like metrics.

Question:

How many payment failures happened?

count_over_time( {service="payment"} |= "failed" [5m] )
Meaning:
Count failed payment logs during last 5 minutes

Error Rate Example

rate( {service="checkout", level="ERROR"} [1m] )
Meaning:
How many checkout errors per second?

Real Production Debugging

Customer says checkout failed
↓
Search trace_id
↓
Find all related logs
↓
Identify payment timeout

Memory Rule

{ } = Select streams using labels |= = contains text != = remove text |~ = regex search | json = extract JSON fields count_over_time = convert logs into numbers rate = logs per second
