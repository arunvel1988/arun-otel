Exporting Logs from Alloy to Loki

Moving application logs into searchable storage

Why Do We Need Loki?

Applications continuously create logs.

Example ecommerce logs:
User added item to cart Checkout started Payment failed Order created

But application containers are temporary. If the container disappears, logs disappear.

We need a central place to store and search logs. That place is Loki.

What is Loki?

Loki is a log storage and query system created by Grafana.

Think of Loki like a database for logs.

Applications create logs
↓
Loki stores logs
↓
Grafana searches logs

Where Export Happens?

Export happens after Alloy collects and processes logs.

Flask Application creates JSON logs
↓
Alloy Receiver collect logs
↓
Processor parse json add labels
↓
Loki Exporter push logs
↓
Loki Storage

What Does Exporter Mean?

Exporter is the output stage.

Its job:

Take processed logs Convert into Loki format Send over HTTP
Simple example:
Alloy has logs
↓
Send HTTP request
↓
Loki receives

Loki Push API

Unlike Prometheus scraping metrics, logs are normally pushed.

Alloy POST http://loki:3100/loki/api/v1/push Loki

Every few seconds Alloy sends collected logs.

Alloy Loki Export Configuration

loki.write "default" { endpoint { url = "http://loki:3100/loki/api/v1/push" } }
Meaning:
Config Meaning
loki.write Create Loki exporter
endpoint Where logs should go
3100 Loki server port

Important Loki Concept: Labels

Loki does not index the entire log message. It indexes labels.

Example log:
Payment failed for order 500
Labels:
service="payment" environment="production" level="ERROR"
Now Grafana searches:
{service="payment"}

What is a Loki Stream?

Logs with the same labels are stored together. This group is called a stream.

Example:
Stream: service="checkout" environment="prod" Logs: checkout started payment started order created
