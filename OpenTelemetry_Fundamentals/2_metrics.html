Metrics Concepts

Understanding Counters, Gauges, Histograms, Exemplars, RED Metrics, and USE Metrics using real-world examples.

What are Metrics?

Metrics are numerical measurements collected over time. They help engineers monitor application and infrastructure health.

  • Measure application performance
  • Track infrastructure health
  • Detect failures and bottlenecks
  • Monitor system behavior
  • Power dashboards and alerts
Examples: CPU Usage = 70% Memory Usage = 4 GB API Requests = 500/sec Checkout Latency = 120 ms

Counter Metrics

Counters only increase over time. They are used to count events.

  • Total API requests
  • Total orders placed
  • Total payment failures
  • Total login attempts

Characteristics

Counter values always increase. They reset only when application restarts.

E-Commerce Example

Total number of checkout requests processed by Checkout API.

checkout_requests_total = 10500 payment_failures_total = 52

Gauge Metrics

Gauges can increase or decrease. They represent current values.

  • CPU Usage
  • Memory Usage
  • Current Active Users
  • Queue Length

Characteristics

Gauge values continuously change depending on system state.

E-Commerce Example

Current number of active users browsing products.

active_users = 320 cpu_usage = 72% inventory_queue_size = 45

Histogram Metrics

Histograms measure value distribution. Commonly used for latency analysis.

  • API response time
  • Database query latency
  • Payment processing duration
  • Request size distribution

Purpose

Helps understand latency distribution across requests.

E-Commerce Example

Checkout API latency distribution analysis.

checkout_latency_bucket{le="100"} 1200 checkout_latency_bucket{le="300"} 3500 checkout_latency_bucket{le="500"} 5200

What are Exemplars?

Exemplars connect metrics with traces. They help engineers jump from a metric directly into a distributed trace.

High API Latency Metric ↓ Associated Trace ID ↓ Open Distributed Trace ↓ Find Slow Service
Concept Description
Metric Numerical measurement
Exemplar Reference to related trace
Trace Complete request journey

RED Metrics

RED metrics focus on application-level monitoring.

Rate

Number of requests handled per second.

Errors

Number of failed requests.

Duration

Request latency and response time.

Checkout API: Rate = 500 requests/sec Errors = 12 failed requests Duration = 120 ms average latency

USE Metrics

USE metrics focus on infrastructure monitoring.

Utilization

Percentage of resource usage.

Saturation

How busy the resource is.

Errors

Infrastructure-level failures.

Kubernetes Node: CPU Utilization = 80% Memory Saturation = High Disk Errors = 5

RED vs USE Metrics

RED Metrics USE Metrics
Focus on applications Focus on infrastructure
Request Rate Resource Utilization
Application Errors Infrastructure Errors
Request Duration System Saturation
API Monitoring CPU, Memory, Disk Monitoring
