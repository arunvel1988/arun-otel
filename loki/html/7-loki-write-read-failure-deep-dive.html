1. Distributor

Distributor is the first Loki component receiving logs.

Responsibilities:
  • Check tenant
  • Validate labels
  • Apply rate limits
  • Select ingester
Example:
Tenant: X-Scope-OrgID = bank-a Labels: service="payment" level="ERROR"

How Distributor Selects Ingester

Labels arrive
|
Create hash
|
Ask Loki Ring
|
Ring returns owner
|
Send to Ingester

2. Ingester

First place where logs are stored.

Storage location: Ingester RAM
Flow:
Receive log
|
Find Stream
|
Create/Open Chunk
|
Append log line

Stream Concept

Same label combination creates one stream.

Example:
tenant=bank-a service=payment level=ERROR
Logs:
10:01 error
10:02 error
10:03 error

Chunk Creation

Chunks are created by:

INGESTER
Open chunk in RAM
|
Collect many logs
|
Close chunk
|
Compress
|
Generate chunk ID
|
Upload MinIO

Why Not Directly Write S3?

1 million logs = 1 million S3 writes

Too expensive.

Instead:
Many logs | One compressed chunk | One upload

PART 5 : FAILURE SCENARIOS

Distributor Down

New logs cannot enter. Alloy retries. Existing queries work.

Ingester Down

Most dangerous. Active RAM chunks are here. Use replication_factor=3.

MinIO Down

Flush fails. Old log queries fail. Ingester retries uploads.

Querier Down

Logs continue. Only searching fails.

Compactor Down

Logs continue. Queries continue. Storage grows. Retention stops.
