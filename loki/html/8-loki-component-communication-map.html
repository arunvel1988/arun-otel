Grafana Loki - Which Component Talks To Which?

The Biggest Loki Misunderstanding

Many people imagine Loki like this:

Distributor | Ingester | Querier | Compactor | Ruler
This is WRONG. Loki components are not one single chain.

Loki has independent flows.

Why Querier Reads Ingester?

Example:
Current time: 10:00:10 Log created: 10:00:05

Only 5 seconds old.

Not flushed to MinIO yet. Still inside Ingester RAM.
So:
Querier
|
Ask Ingester
|
Get fresh logs

Why Querier Reads MinIO?

Older logs are already flushed.

Query Frontend does NOT talk directly to MinIO.

3. BACKGROUND COMPONENTS

Compactor Communication

Compactor is independent.

Compactor
|
MinIO/S3
It performs:
  • Merge index blocks
  • Optimize queries
  • Apply retention
  • Delete expired chunks
Compactor does NOT talk to:

Distributor
Ingester
Querier

Ruler Communication

Ruler behaves like automatic Grafana.

Ruler
|
Run LogQL
|
Querier
|
Result
|
Alertmanager

Failure Impact

Component Failed Impact
Distributor New logs stop. Queries still work.
Ingester Fresh logs affected. Need replication.
Querier Search broken. Ingestion continues.
Query Frontend Queries slower/unavailable.
Compactor Logs work. Retention and cleanup stop.
Ruler Alerts stop. Logs continue.
MinIO Storage and old queries affected.

Final Loki Mental Model

WRITE: Alloy -> Distributor -> Ingester -> MinIO READ: Grafana -> Query Frontend -> Querier -> Ingester + MinIO BACKGROUND: Compactor -> MinIO Ruler -> Querier -> Alertmanager
