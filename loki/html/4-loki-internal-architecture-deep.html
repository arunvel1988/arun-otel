Step 1: Distributor Receives Log

First Loki component that receives logs:

DISTRIBUTOR
Example:
{ tenant:"bank-a", labels:{ service:"payment" }, message: "payment failed" }
Distributor checks:
  • Is tenant allowed?
  • Is log format correct?
  • Rate limits
  • Which ingester should receive?

How Distributor Chooses Ingester?

Using the Loki Ring.

Step 2: Ingester Receives Log

Ingester is the temporary database.

VERY IMPORTANT: First storage location = Ingester RAM
Flow:
New log
|
Find stream
|
Open chunk in memory
|
Append log line

What Is A Stream?

Same labels create one stream.

Example:
service="payment" level="ERROR" tenant="bank-a"
All matching logs go into same stream.

What Is A Chunk?

Chunk is a block of compressed logs.

RAM:

chunk-123
10:01 error
10:02 error
10:03 error

Initially this chunk is NOT in MinIO.

Who Compresses The Chunk?

Answer:
Ingester
When:
  • chunk becomes full
  • chunk_idle_period reached
  • max_chunk_age reached
Flow:
Open Chunk
|
Close Chunk
|
Compress
|
Generate chunk ID
|
Upload to Object Store

Who Creates Random Folder Names?

Answer:
Ingester creates chunk IDs
Example:
MinIO: bank-a/ 74fa83293992/ chunk-file

That ID is based on stream fingerprint and chunk metadata.

What Happens After Upload?
