SCENARIO 1

Sudden Log Explosion

Example:
Normal: 1 million lines/min Problem: 50 million lines/min

What Happens Internally?

Huge traffic
|
Distributor receives
|
Checks limits
|
Too much?
|
Reject
Error:
HTTP 429 Too Many Requests

How Admin Detects?

Prometheus:
rate( loki_distributor_lines_received_total[5m] )
Rejected logs:
loki_discarded_samples_total

How To Fix?

Distributor is stateless. Scale it:
kubectl scale deployment loki-distributor \ --replicas=10
Before:
Distributor-1
After:
Distributor-1 Distributor-2 Distributor-3 ... Distributor-10

SCENARIO 2

Ingester Memory Explosion

Symptoms:
Pod OOMKilled High RAM Slow ingestion

Why Does Ingester Memory Increase?

Remember:
Ingester stores ACTIVE chunks in RAM
Main reason:

High Cardinality Labels

Bad:
user_id="983747" request_id="abc123" transaction_id="777"
Creates:
Millions of streams
|
Millions of chunks
|
Huge RAM

How To Detect?

Metrics:
loki_ingester_memory_streams process_resident_memory_bytes

How To Fix?

1. Remove bad labels in Alloy

Remove: request_id user_id session_id

2. Increase ingesters

kubectl scale statefulset loki-ingester \ --replicas=20

SCENARIO 3

Ingester Crash

Why dangerous?
Fresh logs live in Ingester RAM
Example:
10:00 log arrives
|
RAM chunk
|
Not yet S3
|
Crash

Production Protection 1: Replication

Config:
replication_factor: 3
Meaning:
Distributor
|
+-----------+-----------+
|
Ingester-1
Ingester-2
Ingester-3
If one dies:
Two copies remain

Production Protection 2: WAL

Write Ahead Log.
Incoming log
|
Write local disk WAL
|
RAM chunk
Crash recovery:
Restart
|
Replay WAL
|
Recover chunks

SCENARIO 4

S3 / MinIO Failure

Flow:
Distributor OK
|
Ingester OK
|
S3 FAILED
Symptoms:
Chunk flush failures
Metric:
loki_ingester_chunk_store_failures_total

Fix

  • Recover object storage
  • Check network
  • Check credentials
  • Ensure WAL has space

Write Path Troubleshooting Checklist

Problem Check
No logs Alloy → Distributor → Ring → Ingester
429 errors Tenant limits
High RAM Ingester streams/cardinality
Flush errors S3 connection
Data loss risk Replication + WAL
