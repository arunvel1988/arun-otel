SCENARIO 5

Grafana Queries Are Very Slow

User complaint:
Dashboard keeps loading Explore query timeout
First rule:
Do not restart Loki randomly. Find the bottleneck.

Step 1: Check Query Frontend

Role:
  • Receives big queries
  • Splits queries
  • Queues work
  • Uses cache
Example:
User query: Last 24 hours ERROR logs
Query Frontend:
24 hour query
|
split
|
0-6 hours
6-12 hours
12-18 hours
18-24 hours

Query Frontend Problem

Symptom:
Large queue Many waiting queries
Metric:
loki_query_frontend_queue_length
Fix:
kubectl scale deployment loki-query-frontend \ --replicas=5

SCENARIO 9

Compactor Problems

Remember:
Compactor does not handle live logs.
Its job:
  • Merge index blocks
  • Optimize storage
  • Delete expired chunks

If Compactor Dies

Logs continue Queries continue BUT Retention stops Storage grows
Metric:
loki_compactor_runs_completed_total
Fix:
kubectl rollout restart deployment loki-compactor

Production Scaling Guide

Problem Scale / Fix
Too many incoming logs Distributor
High memory usage Ingester + fix labels
Slow Grafana queries Querier
Large query pressure Query Frontend
Storage increasing Compactor + retention
Alerts delayed Ruler
Huge S3 cost Alloy filtering

Loki SRE Troubleshooting Formula

WRITE ISSUE? Check: Alloy Distributor Ring Ingester S3 QUERY ISSUE? Check: Query Frontend Querier Cache Index S3 COST ISSUE? Check: Retention Labels Log volume Compactor

Final Production Memory

Distributor = Can Loki accept traffic? Ingester = Can Loki safely store incoming logs? S3 = Can Loki keep history? Querier = Can users search? Frontend = Can Loki handle big queries? Compactor = Can Loki control storage? Ruler = Can Loki alert automatically?
