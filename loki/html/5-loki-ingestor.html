Operating Loki In Production

Distributor
Ingester
Querier

loki-compactor Deployment
loki-ruler Deployment

2. Distributor Administration

Responsibility

First Loki component receiving logs
Tasks:
  • Authenticate tenant
  • Validate labels
  • Check rate limits
  • Select ingester
Flow:
Alloy
|
Distributor
|
Hash labels
|
Find ingester

3. Scaling Distributor

Problem:

Too many logs arriving
Solution:
Increase distributor replicas
Example:
Before: Distributor-1

After: Distributor-1 Distributor-2 Distributor-3

Distributor is stateless. Easy to scale.

4. Ingester Administration

Most important Loki component.

Responsibilities:
  • Create streams
  • Create chunks
  • Hold chunks in RAM
  • Compress chunks
  • Upload to S3
Flow:
Log
|
RAM chunk
|
Compress
|
S3

5. Why Ingester Is Stateful?

Because:
It owns active chunks in memory
Therefore Kubernetes uses:
StatefulSet not Deployment
Example:
loki-ingester-0 loki-ingester-1 loki-ingester-2

6. Ingester Failure

Scenario:
ingester-1 crashes
If:
replication_factor:1
Result:
Unflushed memory chunks may disappear
Production:
replication_factor:3
Flow:
Same log stored: Ingester-1 Ingester-2 Ingester-3

7. WAL - Write Ahead Log

Protection against ingester crash.

Flow:
Incoming Log
|
Write WAL on disk
|
Keep RAM chunk
If crash:
Restart | Read WAL | Recover chunks

8. Query Frontend

Protects queriers.

Without frontend:
Grafana | Querier overloaded
With frontend:
Grafana | Query Frontend | Split query | Queue | Querier

9. Querier Administration

Responsibility:
  • Read index
  • Find chunks
  • Download from S3
  • Decompress logs
Flow:
Query: ERROR logs
|
Index lookup
|
Find chunks
|
Download S3

10. Query Cache

Problem:
Same dashboard refreshes every 10 seconds
Without cache:
Every query hits S3
With cache:
Grafana | Cache hit | Return result
Production uses:
memcached

11. Ruler Component

Ruler runs LogQL automatically.

Example:
If ERROR count > 100 send alert
Flow:
Ruler | Runs LogQL | Alertmanager | Notification

12. Compactor Operations

Responsibilities:
  • Compact indexes
  • Apply retention
  • Delete old chunks
Failure:
Logs continue Queries continue Storage keeps growing

13. Loki Monitoring Metrics

Admin watches:
loki_distributor_lines_received_total loki_ingester_memory_streams loki_ingester_chunks_flushed_total loki_request_duration_seconds loki_cache_hits_total

14. Troubleshooting Examples

No logs arriving

Check:
Alloy Distributor Tenant header Rate limits

High Memory

Check:
Ingester Number of streams Bad labels Cardinality

Slow Queries

Check:
Query frontend Querier replicas Cache S3 latency

Loki Admin Mental Model

Distributor = traffic controller Ingester = temporary database + compressor S3 = permanent storage Querier = log reader Frontend = query manager Compactor = cleanup worker Ruler = alert engine
