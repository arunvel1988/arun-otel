Loki + MinIO: How Logs Are Really Stored

What You See In MinIO

Example:
loki-data | + bank-a | + 4f6e7a9c + 82ab9210 + index_20345

These random looking names are created by Loki. They are not created by MinIO.

Who Creates These Random Folders?

Answer:
Loki Ingester
Flow:
Log arrives
|
Ingester creates chunk in memory
|
chunk becomes full / old
|
compress chunk
|
generate unique chunk ID
|
upload object to MinIO
Example chunk name:
bank-a/1874af92c93e238
Meaning:
bank-a = tenant 1874af92 = unique chunk fingerprint

What Is Inside A Chunk?

A chunk contains many log lines. Example original logs:
10:00 payment success 10:01 payment failed 10:02 user checkout
Loki compresses:
1000 log lines | compression | one chunk file
The chunk contains:
  • timestamps
  • actual log messages
  • compressed data

Where Are Labels Stored?

Labels do not live inside the chunk search path. Example:
service="payment" level="ERROR"
go into:
TSDB Index
Index works like:
service=payment | v chunk id 87472

When Grafana Queries What Happens?

Important:
Grafana NEVER talks directly to MinIO.
Actual flow:
Grafana
|
LogQL Query
|
Loki Query Frontend
|
Querier
|
Read Index
|
Find matching chunks
|
Download chunks from MinIO
|
Return logs

Example Query Internals

User runs:
{service="payment"}
Loki does:
Step 1: Search index Find: chunk-111 chunk-222 Step 2: Get from MinIO Step 3: Decompress Step 4: Return matching logs

Does Loki Read Entire MinIO Bucket?

No. Example:
MinIO: 10 TB logs Query: last 5 minutes ERROR
Loki reads:
Only required chunks Maybe few MB
Because index tells location.

Can I Decide What Goes To MinIO?

YES. But usually NOT inside Loki. Production filtering happens BEFORE Loki.

Filtering Location 1: Alloy (Best)

Example: Do not store DEBUG logs.
Application | Alloy | (drop DEBUG) | Loki | MinIO
Example Alloy rule:
stage.drop { expression = ".*DEBUG.*" }
Result:
DEBUG never reaches Loki DEBUG never reaches MinIO

Filtering Location 2: Loki Limits

Loki can reject based on rules:
limits_config: reject_old_samples:true ingestion_rate_mb:10
Used for protection. Not normal filtering.

Should All Logs Go To MinIO?

Production answer: NO. Example:
Keep: ERROR WARN Business events Reduce: DEBUG Health checks Noise logs

Final Interview Answer

MinIO does not organize logs. Loki creates tenants, indexes, chunk IDs. Grafana never reads S3. Queries always go: Grafana -> Loki -> MinIO Filtering should happen in Alloy before storage.
